# Proposal: Internationalization of The Latent Review

**Status: proposal only — nothing here is built.** Item 7 of the editors'
2026-07-16 batch asked for a design the editors can accept, amend, or
decline before any implementation PR is opened. Costs and mechanics below
are estimates to be re-baselined at build time.

## 1. What gets localized, and how

Three tiers, by how much we stand behind the words:

| Tier | Content | Method | Reviewed? |
|------|---------|--------|-----------|
| 1 | UI chrome (nav, footer, labels, form text) | Translated once per string change, by Claude, committed to the repo | Yes — small enough for editor spot-review in the PR |
| 2 | Charter, About, standing pages | Full-document machine translation, committed to the repo | Labeled; **English remains the binding text** — each translation opens by saying so |
| 3 | Articles and correspondence | Machine translation at publish time, batch, committed to the repo | **No.** Labeled as automated and unreviewed |

Every tier-2 and tier-3 page carries a visible label, itself localized,
naming the model and stating the ruling's required text:

> Translated by Claude Opus 4.8 (claude-opus-4-8). **English is canonical;
> this translation is automated and unreviewed by the editors.** [Read the
> original →]

Nothing about this touches provenance rules: the provenance block's **label
value is quoted verbatim in English** in every locale (it is the record, and
the record is never altered — only the field names around it localize), with
a localized note pointing at the canonical English page.

## 3. Astro approach

Astro's built-in i18n routing fits a fully static build:

```js
// astro.config.mjs
i18n: {
  defaultLocale: 'en',
  locales: ['en', 'es', 'fr', /* … */ 'he', 'ar', 'ur'],
  routing: { prefixDefaultLocale: false },
}
```

- **UI strings (tier 1):** a typed dictionary per locale in `src/i18n/`
  (`en.ts` is the source of truth; TypeScript fails the build if a locale is
  missing a key). Components read strings via a `t(locale, key)` helper.
- **Governance docs (tier 2):** `docs/i18n/CHARTER.<tag>.md` etc., rendered
  by the same verbatim-from-repo pattern as `/charter` today.
- **Articles (tier 3):** translated Markdown lives in the content
  collection as siblings, e.g. `src/content/articles/<slug>/index.md`
  (English) plus `<slug>/<tag>.md`, sharing the English entry's frontmatter
  (provenance is inherited, never re-stated per locale) with added
  `translation` metadata: model ID, source commit of the English text, date.
- **Committed, not build-time:** translations are generated by a script and
  committed through a PR — never fetched or generated during `astro build`.
  The public git history then proves what each translation said and when,
  the same provenance argument the journal already makes; a build costs $0
  in tokens; and a bad translation is revertable.

## 4. URLs

- English stays at the root — every existing URL is unchanged and canonical.
- Locales get a path prefix: `/es/articles/<slug>/`, `/ar/charter/`.
- **Slugs stay English** in every locale: stable identifiers, one slug per
  piece across 20 languages.
- Every page emits `hreflang` alternates for all locales it exists in, plus
  `x-default` → English; `rel=canonical` on a translation points to itself
  (it is the canonical *for that language*), and the sitemap carries the
  alternate links. Feeds (`rss.xml`, `feed.json`) remain English-only at
  launch; per-locale feeds are a later decision.

## 5. RTL design (Hebrew, Arabic, Urdu)

- `<html lang="{tag}" dir="rtl">` on the three RTL locales; all others LTR.
- **Logical-properties audit:** the codebase already leans logical
  (`margin-inline`, `padding-inline`), but a handful of physical properties
  must convert so the layout mirrors correctly:
  `blockquote { padding-left; border-left }` → `padding-inline-start;
  border-inline-start`, table `text-align: left` → `text-align: start`,
  figcaption/cover-caption `text-align: right` → `text-align: end`. The
  double rules, centered mastheads, and centered nav are direction-neutral.
- **Provenance blocks in RTL** (the ruling's named concern): field names
  (dt) localize and align to the start edge; the grid uses logical columns
  so it mirrors automatically. Values that are Latin-script records — the
  verbatim English provenance label, model IDs like `claude-fable-5`,
  dates, email-like strings — are wrapped in `<bdi>`/`dir="ltr"` spans with
  `unicode-bidi: isolate` so bidi reordering can never scramble punctuation
  inside the record. The block stays mono-faced: IBM Plex Mono lacks
  Hebrew/Arabic glyphs, so the mono stack gains per-script fallbacks
  (below) while Latin record-values still render in Plex.
- **Typography:** EB Garamond / Source Serif 4 cover Latin, Greek, and
  Cyrillic only. Per-script self-hosted font subsets, loaded only on pages
  of that locale (so the English site's payload is unchanged): Noto Serif
  Hebrew, Noto Naskh Arabic (shared by `ar` and `ur`; optionally Noto
  Nastaliq Urdu for `ur` if the editors prefer the traditional style —
  larger and line-height-hungry, flagged as a taste decision), Noto Serif
  Devanagari/Bengali/Telugu/Tamil/Gurmukhi, Noto Serif KR/JP/SC (CJK subsets
  are large; subset aggressively and lean on system serif fallbacks).
  Letterspaced-uppercase styles (kickers, nav) apply only to scripts that
  have case; the style falls back to plain weight elsewhere — this keeps
  the journal's character without faking it in scripts where it doesn't
  exist.

## 6. Translation workflow

A batch script in the house pattern (`scripts/send-issue.mjs`): manual,
dry-run by default, hard-capped per run, loud about what it will spend.

1. On publish (dual-yes, English), run `node scripts/translate.mjs` — it
   finds published English pieces lacking translations, up to a hard cap.
2. Submits one job per (piece × language) to the **Batches API** (50%
   price, fully async — right shape for a nightly/weekly cadence), with a
   translation prompt that forbids adding, removing, or annotating content
   and echoes the reader-protection clause (no injected directives).
3. Writes the returned Markdown into the content tree with translation
   metadata, opens a PR. Editors merge like any other PR.
4. **Corrections:** when the English text changes (visible correction), the
   script detects the stale source-commit hash and re-translates; the old
   translation stays in git history.

Cost-guardrail properties, consistent with the standing rules: reader
traffic can never trigger a token spend (translations are static files);
the script is manual/scheduled with a hard cap; submissions still trigger
nothing.

## 7. Costs

Assumptions to argue with: ~5 pieces/week averaging ~2,500 words; ~1.35
tokens/word English (~3,400 tokens in, ~1.5× out — non-Latin scripts often
tokenize heavier); ~4k input tokens per job with prompt overhead; 19 target
languages ⇒ ~95 jobs/week ≈ 0.38M input + 0.48M output tokens weekly.

Weekly, at Batches API pricing (50% of list):

| Model | $/MTok in/out (list) | Weekly | Yearly |
|-------|----------------------|--------|--------|
| Haiku 4.5 | $1 / $5 | ~$1.40 | ~$75 |
| Sonnet 5 | $3 / $15 (intro $2/$10 to 2026-08-31) | ~$4.15 (~$2.75 intro) | ~$215 |
| Opus 4.8 | $5 / $25 | ~$6.90 | ~$360 |
| Fable 5 | $10 / $50 | ~$13.80 | ~$715 |

Even the top of the table is small next to the editorial stakes, so the
**recommendation is quality-first: Claude Opus 4.8** for articles (a
literary journal should not read like machine output in any language), with
the one-time tier-1/tier-2 set done by the same model. Whichever model is
chosen, the ruling's label names it, so switching models later is visible
by design. Correspondence (≤500 words) adds roughly 10–20% to the totals if
translated; the editors may also rule letters English-only.

One-time: charter + about + UI strings ≈ 6k words ⇒ under $5 at Opus
Batches pricing for all 19 languages.

## 8. Build and operational impact

- Page count multiplies by ~20: today's 13 pages → ~260, plus 20 pages per
  published article. Static builds at this scale are still fast (seconds to
  low minutes) and Netlify-friendly; revisit only if the archive grows to
  thousands of pieces.
- Font payload is per-locale (see §5); English readers download nothing new.
- `llms.txt` gains a short i18n section stating locales, the canonical-
  English rule, and the URL scheme — machine readers get the same honesty
  as human ones.

## 9. Proposed phasing

- **Phase 1 (pilot, 3 locales):** Spanish, French, **Hebrew** — two
  high-coverage Latin locales plus one RTL to prove the hard rendering path
  (bidi provenance blocks, mirrored layout, fonts) before it's multiplied.
- **Phase 2:** the remaining 16, mechanical once Phase 1's pipeline and RTL
  patterns are settled.
